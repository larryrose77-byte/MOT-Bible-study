> This post exists to show the layout. The video, audio, and transcript above are a short clip from a financial webinar — **not** Bible study content. They're here only to demonstrate that the publishing pipeline (Zoom recording -> R2 upload -> Whisper transcript -> Word document download) works end-to-end. Replace this post with your first real Monday study.

## How posting a real study works

For each Monday's study:

1. **Record** the discussion via Zoom (local recording).
2. **Transcribe** the audio:
   ```sh
   npm run transcribe -- "C:\path\to\audio1234567.m4a"
   ```
   Produces a `.docx` next to the audio file.
3. **Upload** all three assets to R2:
   ```sh
   npm run audio:upload      -- "C:\path\to\audio1234567.m4a"
   npm run video:upload      -- "C:\path\to\video1234567.mp4"
   npm run transcript:upload -- "C:\path\to\audio1234567.docx"
   ```
   Each command prints the public URL.
4. **Create the Markdown post** in `src/content/studies/` (e.g. `2026-06-23-genesis-1-6-13.md`) with the three URLs in the frontmatter (copy this file as a template).
5. **Publish:** `npm run deploy`.

## Why this approach

- Video and audio live in **Cloudflare R2** (10 GB free, no egress fees) — too large to ship through Cloudflare Pages directly.
- Transcription runs **locally** via OpenAI's Whisper (free, no API key, no per-minute cost). The `base.en` model gives decent quality; bump to `--model=small.en` or `--model=medium.en` for better accuracy at the cost of slower processing.
- The Word document is generated by `scripts/transcribe.mjs` using the `docx` npm package — paragraph-broken and headed, ready to share.
